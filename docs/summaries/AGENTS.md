# Executive Summary Format

Create one summary for each qualifying implementation task using the shared identifier defined in
`docs/AGENTS.md`.

## Required Sections

1. **Outcome** — what is now true in plain language.
2. **User Impact** — what Anthony can see, do, or trust that changed.
3. **MS-01 Progress** — the user-specified phase and objective advanced.
4. **Key Decisions** — only consequential implementation choices.
5. **Limits and Next Gate** — deferred work, known constraints, and the next roadmap gate.

Keep the summary brief enough to scan without reading code. Link the matching test report and
walkthrough near the top. Do not repeat detailed test logs or turn the summary into a changelog.
